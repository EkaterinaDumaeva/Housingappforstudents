• Serious
• Supportive
• Calm
• Human-centered

• More visible
• Larger
• More noticeable
• Clearly emergency-oriented
• Mobile-first

Suggested labels:
• Emergency Help
• Need Urgent Help?
• Unsafe Situation?
• Get Help Now

Avoid:
• Tiny support links
• Generic “Customer Support” wording
• Corporate/helpdesk feeling

CHATBOT FIRST RESPONSE

Keep the existing chat system but redesign the first support interaction into a smart emergency intake chatbot.

The bot should:
• Calmly collect information
• Ask important safety questions
• Escalate urgent cases to Admin

The bot should feel:
• Human
• Calm
• Helpful
• Non-judgmental
• Fast

Avoid robotic/corporate wording.

Example opening:
“Hi. I’m here to help. Are you currently experiencing a problem or unsafe situation?”

The chatbot should collect:
• Emergency type
• Severity
• Description
• Screenshots/photos/videos
• Whether user feels unsafe
• Whether urgent relocation/help is needed
• User location placeholder
• Contact information
• Booking/employer connection if available

Create UI for:
• Upload screenshots
• Upload photos/videos
• Share location placeholder
• Voice note placeholder optional
